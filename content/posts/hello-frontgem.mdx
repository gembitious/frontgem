## 왜 또 블로그인가

세상에 블로그 플랫폼은 많다. 그럼에도 직접 만드는 이유는 하나다.
**퇴고 과정 자체를 도구로 만들고 싶어서**다. 초안을 원석이라 부르고,
다듬어 발행하는 과정을 세공(lapidary)이라 부르는 이유다.

## 저장소가 곧 데이터베이스

이 블로그에는 데이터베이스가 없다. 글은 전부 `content/posts/` 아래의
MDX 파일이고, 발행이란 곧 이 디렉터리에 커밋하는 행위다.

- git 히스토리가 곧 버전 관리이자 백업이다.
- 빌드 시점에 [Velite](https://velite.js.org)가 MDX를 타입 안전한
  데이터 레이어로 컴파일한다.
- 배포는 Vercel이 커밋을 감지해 정적으로 재생성한다.

```ts
// 발행은 결국 파일 하나를 커밋하는 일이다.
async function publish(post: Post): Promise<void> {
  const mdx = serialize(post.blocks)
  await github.commit(`content/posts/${post.slug}.mdx`, mdx)
  await revalidate(`/posts/${post.slug}`)
}
```
