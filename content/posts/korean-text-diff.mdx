## 공백은 단어 경계가 아니다

영어 diff는 공백으로 토큰을 나누면 대체로 맞다. 한국어는 그렇지 않다.
조사가 어절에 붙고, 띄어쓰기 규칙은 유동적이며, 같은 의미가 붙여쓰기와
띄어쓰기로 모두 표현된다.

> "간결하게 고쳐줘"와 "간결하게고쳐줘"는 사람에겐 같지만
> 공백 토큰화에겐 전혀 다른 입력이다.

## Intl.Segmenter로 어절을 나눈다

브라우저 내장 `Intl.Segmenter`는 로케일 기반 분절을 제공한다.

```ts
const seg = new Intl.Segmenter('ko', { granularity: 'word' })
const tokens = [...seg.segment('간결하게 고쳐줘')]
  .filter((s) => s.isWordLike)
  .map((s) => s.segment)
// → ['간결하게', '고쳐줘']
```

이렇게 얻은 어절 토큰 위에서 LCS 기반 정렬을 돌리면, 블록 내 단어 단위
diff가 한국어에서도 의미 있는 hunk로 떨어진다.

## 블록 정렬이 먼저다

문서는 블록 배열이다. 먼저 블록 단위로 LCS 정렬을 한 뒤, 짝지어진 블록
내부에서만 어절 diff를 계산한다. 이 2단계 구조가 diff를 안정적으로 만든다.
